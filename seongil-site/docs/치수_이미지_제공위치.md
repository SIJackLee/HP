# 치수 이미지 자료 제공 위치

팬(A/B/C/D/E), 후드(A~F), 셔터(B/A/T), 인렛(정면 외형·개구부·상단·측면·두께) 치수에 대한 이미지 자료의 **저장 경로**와 **화면 표시 위치**를 정리했습니다.

---

## 1. 파일 저장 위치 (제공하실 이미지 넣는 곳)

| 구분 | 폴더 경로 | 권장 파일명 | 비고 |
|------|-----------|-------------|------|
| **팬 치수** | `assets/images/dimensions/` | `fan-dimensions-a.png` (A타입 도면) | A타입 공통 도면 1장 |
| | | `fan-dimensions-d.png` (D타입 도면) | D타입 공통 도면 1장 |
| **후드 치수** | `assets/images/dimensions/` | `hood-dimensions.png` | 후드 A,B,C,D,E,F 표기 도면 |
| **셔터 치수** | `assets/images/dimensions/` | `shutter-dimensions.png` | 셔터 B, A, T 표기 도면 |
| **인렛 치수** | `assets/images/dimensions/` | `inlet-dimensions.png` | 정면 외형 가로/세로, 개구부 가로/세로, 상단 표기, 측면 깊이, 측면 하단 표기, 두께 표기 도면 |

- **공통**: 프로젝트 내 `assets/images/dimensions/` 폴더에 위와 같이 저장하면, 제품소개 페이지에서 “치수” 영역과 함께 사용할 수 있습니다.
- 확장자는 `.png` 또는 `.jpg` 등 실제 사용 형식에 맞춰 적용하면 됩니다.

---

## 2. 화면 표시 위치 (제품소개 페이지 내)

이미지는 **해당 치수 카드가 나오는 블록 바로 위 또는 왼쪽**에 두는 구성을 권장합니다.

| 치수 종류 | 표시 위치 (페이지 구조) | 해당 HTML 영역 ID |
|-----------|------------------------|-------------------|
| **팬 치수 A/B/C/D/E** | 제품소개 → **환기팬** 탭 → 형식 선택 후, “치수 (A)” / “치수 (D)” **카드 그리드 위** | `#fanDimCards` 위에 이미지 블록 추가 |
| **후드 치수 A~F** | 제품소개 → **후드** 탭 → TYPE 선택 후, 스펙 카드 **위** | `#hoodSpecCards` 위에 이미지 블록 추가 |
| **셔터 치수 B/A/T** | 제품소개 → **셔터** 탭 → TYPE 선택 후, 스펙 카드 **위** | `#shutterSpecCards` 위에 이미지 블록 추가 |
| **인렛 치수** (정면 외형 가로, 정면 외형 세로, 개구부 가로, 개구부 세로, 상단 표기, 측면 깊이, 측면 하단 표기, 두께 표기) | 제품소개 → **인렛** 탭 → 카드 리스트 **위** | `#inletSpecCards` 위에 이미지 블록 추가 |

- “치수 카드”는 그대로 두고, 그 **위쪽**에 치수 도면 이미지 1장을 배치하는 방식이 자연스럽습니다.
- 모바일에서는 이미지를 카드 그리드 위에 세로로 배치하면 됩니다.

---

## 3. 적용 방식 제안

1. **이미지 파일**  
   - 위 표대로 `assets/images/dimensions/`에 저장 (파일명은 표와 동일하거나, 팀 내 규칙에 맞게 통일).

2. **표시 규칙**  
   - **팬**: `#fanDimCards` 영역 렌더 시, “치수 (A)” / “치수 (D)” 제목 **위**에  
     `fan-dimensions-a.png` / `fan-dimensions-d.png`를 한 번씩만 노출.  
   - **후드**: `#hoodSpecCards` **위**에 `hood-dimensions.png` 노출.  
   - **셔터**: `#shutterSpecCards` **위**에 `shutter-dimensions.png` 노출.  
   - **인렛**: `#inletSpecCards` **위**에 `inlet-dimensions.png` 노출.

3. **구현 옵션**  
   - **A) 정적**: `products.html`에 각 탭 패널 안에 `<img>` 블록을 넣고, CSS로 해당 ID 위에 배치.  
   - **B) 동적**: `product-intro.js`에서 치수/스펙 영역을 그릴 때, 같은 블록 **앞**에 `<img src="assets/images/dimensions/xxx.png" alt="치수 도면">`을 삽입.

원하시면 A 또는 B 중 하나를 정해서, `products.html` / `product-intro.js` 수정 예시까지 구체 코드로 작성해 드리겠습니다.
