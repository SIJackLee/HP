# 제품소개에 재원표 적용 방법

## 1. 현재 구조

- **content.js** `products[]`: 제품별 `name`, `description`, `imageFileName`, `specs{}`
- **제품소개 페이지**: 카드 클릭 시 모달에서 `specs`를 키-값 목록으로 표시
- **재원표**: 제품_재원표.xlsx (SLB, SLF 1P/3P, Controllers, Hood, Shutter, Inlet 시트)

---

## 2. 적용 방안 (3가지)

### 방안 A: content.js에 재원표 요약 반영 (권장·즉시 적용)

**개요**: 재원표 시트별로 **대표 스펙만** 뽑아서 기존 `products[].specs`를 갱신한다.

| 제품 카드 | 재원표 시트 | content.js specs 예시 |
|-----------|-------------|------------------------|
| 환기용 배기휀 | SLB_Motor_Specs, SLF_1P, SLF_3P | 모델: SLB/SLF 시리즈, 전압: 220V/380V, 풍량: CMH, 인치 등 |
| 휀 컨트롤러 | Controllers | 모델: SL10~SL805, 정격전류, 채널, 타이머 등 |
| 악세사리 | Hood_Sizes, Shutter_Sizes | 후드: SLH-350/500/630, 샷터: SLS-35~96 |

**작업 순서**
1. 재원표에서 시트별 대표 행·컬럼만 정리(예: SLB 1행, SLF 대표 2~3행, 컨트롤러 2~3행).
2. `content.js`의 해당 제품 `specs`를 그 내용으로 교체(문자열 요약 또는 키-값).
3. 필요 시 제품 카드 수/이름을 재원표 구성에 맞게 조정(예: "SLB(BLDC)", "SLF 오토휀" 등).

**장점**: 수정만으로 적용, 빌드 없음.  
**단점**: 엑셀 수정 시 수동으로 content.js 동기화 필요.

---

### 방안 B: 엑셀 → JSON 변환 후 사이트에서 사용

**개요**: 제품_재원표.xlsx를 스크립트로 읽어 **JSON 파일**로 저장하고, 제품소개 페이지/모달에서 이 JSON을 참조한다.

**작업 순서**
1. `scripts/`에 `xlsx_to_spec_json.py` 추가:  
   - 시트별로 읽어 `specs/slb.json`, `specs/slf_1p.json`, `specs/controllers.json` 등 생성.
2. `content.js`의 `products[]`에는 `specSource: "specs/slb.json"`처럼 파일명만 두거나,  
   또는 `app.js`에서 fetch로 해당 JSON 로드 후 모달에 표시.
3. 재원표 수정 시 스크립트만 다시 실행해 JSON 갱신.

**장점**: 재원표가 단일 소스, 시트 전체를 그대로 활용 가능.  
**단점**: 배포 전 변환 단계 필요, JSON 구조·로딩 방식 설계 필요.

---

### 방안 C: 제품 카테고리 재정의 + 시트별 상세 표시

**개요**: 제품소개 카드를 재원표 시트 구조에 맞춰 **카테고리별**로 나누고, 모달에서 해당 시트 데이터를 **테이블**로 보여준다.

**카드 예시**
- SLB (BLDC 모터)
- SLF 단상 220V
- SLF 3상 220/380V
- 컨트롤러
- 후드 / 샷터

**작업 순서**
1. `content.js`의 `products`를 위 카테고리로 재구성(이름, 설명, 이미지, `specSheetId` 등).
2. 재원표 시트별 데이터를 JSON 또는 content.js 내 객체로 보관(방안 B와 유사).
3. 모달에서 `specSheetId`에 따라 해당 시트 데이터를 테이블(행/열)로 렌더링.
4. (선택) "상세는 카탈로그 PDF 참조" 링크 유지.

**장점**: 재원표와 제품소개 구조를 1:1로 맞출 수 있음.  
**단점**: 카드/모달/데이터 구조 변경이 커짐.

---

## 3. 방안 A 상세 예시 (content.js 수정)

재원표를 요약해 기존 형식에 맞추는 예시만 적는다.

```javascript
// 예: 환기용 배기휀 (SLB + SLF 요약)
{
  name: "환기용 배기휀",
  description: "SLB(BLDC), SLF(AC) 단상/3상. 다양한 사이즈.",
  imageFileName: "slf-autofan.png",
  specs: {
    "모델(SLB)": "SLB-350R6 ~ 960R5",
    "모델(SLF)": "SLF-200D2-6 ~ 960A8-5 등",
    "입력전압": "220V (단상), 220V/380V (3상)",
    "풍량(CMH)": "1,698 ~ 20,400 (형식별 상이)",
    "인증": "CE 인증 (AC), CE/KC 인증 (BLDC)"
  }
}

// 예: 휀 컨트롤러 (Controllers 시트 요약)
{
  name: "휀 컨트롤러",
  description: "SL10~SL805. 온도/타이머/릴레이 제어.",
  imageFileName: "ventilation-controller.png",
  specs: {
    "모델": "SL10, SL200, SL300, SL800~SL805 등",
    "정격전류": "5A~20A",
    "기능": "팬속도 제어, 타이머, 온도보정, 경보 등"
  }
}
```

이렇게 하면 **지금 사용 중인 모달(키-값 목록)** 이 그대로 재원표 기반 스펙을 보여준다.

---

## 4. 정리

| 구분 | 방안 A | 방안 B | 방안 C |
|------|--------|--------|--------|
| 난이도 | 낮음 | 중간 | 높음 |
| 재원표 반영 | 요약 수동 입력 | 시트→JSON 자동 | 시트별 테이블 |
| 유지보수 | 엑셀 변경 시 수동 | 스크립트 재실행 | 구조 변경 시 코드 수정 |

- **빠르게 적용**하려면 **방안 A**로 content.js의 `specs`만 재원표에 맞게 고치는 것을 권장한다.
- **재원표를 단일 소스로 두고** 자동 반영하려면 **방안 B**를 도입하는 순서를 추천한다.

원하는 방안을 정해주시면, 그에 맞춰 `content.js` 수정 예시나 스크립트/JSON 구조까지 구체적으로 작성해 드리겠습니다.
