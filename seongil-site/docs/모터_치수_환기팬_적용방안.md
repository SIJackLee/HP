# 모터(SLB) 치수 표기·이미지 적용 방안

**전제**: 모터(SLB-350 등)는 동일 규격의 환기팬(SLF-350 등) 치수를 따른다.

---

## 1. 현재 상태

| 구분 | 내용 |
|------|------|
| **SLB-960** | 전용 치수 시트 `SLB960_Dim`(높이(mm), 지름(mm))만 존재 → 치수 카드 + `SLB_A_Size.png` 표시 |
| **그 외 모터** (SLB-350R6, 400R3, 450R6, 500, 600, 630, 730) | 치수 시트 없음 → 치수 카드·이미지 미표시 |

---

## 2. 적용 원칙

- **SLB-960**: 기존 유지. `SLB960_Dim` + `SLB_A_Size.png`.
- **그 외 SLB-xxx**: 모델명에서 **규격 숫자**를 추출한 뒤, 해당 숫자의 **환기팬 치수**(SLF_A_Dimensions / SLF_D_Dimensions)를 사용하고, 동일 규칙으로 **치수 이미지**(SLF_A_Size.png / SLF_D_Size.png)를 적용한다.

---

## 3. 모터 → 환기팬 치수 매핑

### 3.1 규격 숫자 추출

- 모터 모델: `SLB-350R6`, `SLB-400R3`, `SLB-960R5` 등.
- 정규식: `SLB-(\d+)` → 추출 숫자 **350, 400, 450, 500, 600, 630, 730, 960**.

### 3.2 사용할 치수 데이터

| 추출 숫자 | SLF_A_Dimensions (TYPE) | SLF_D_Dimensions (TYPE) | 비고 |
|-----------|-------------------------|--------------------------|------|
| 350 | SLF-350A | SLF-350D | A·D 둘 다 존재 |
| 400 | SLF-400A | SLF-400D | A·D 둘 다 존재 |
| 450 | SLF-450A | — | A만 존재 |
| 500 | SLF-500A | — | A만 존재 |
| 600 | SLF-600A | — | A만 존재 |
| 630 | SLF-630A | — | A만 존재 |
| 730 | SLF-730A | — | A만 존재 |
| 960 | **(모터 전용)** | — | `SLB960_Dim` 사용, 이미지 `SLB_A_Size.png` |

- **960**: 모터 탭에서는 **SLB960_Dim**만 사용하고, 환기팬 치수(SLF-960A)는 사용하지 않음.
- **350, 400**: A·D 둘 다 있으므로, 정책에 따라 **A만 / D만 / A+D 둘 다** 표시 중 하나 선택 필요(아래 4.2 참고).

---

## 4. 치수 표기 방안

### 4.1 카드 표기

- **SLB-960**  
  - 기존: `치수` 제목 하위에 **높이(mm), 지름(mm)** 카드.
- **SLB-350 등 (환기팬 치수 사용)**  
  - 환기팬 탭과 동일한 구조:
    - **치수 (A)** 제목 + A, B, C(타공), D, E 카드 (SLF_A_Dimensions 행 사용)
    - **치수 (D)** 제목 + 날개직경(mm), A, B, C 카드 (SLF_D_Dimensions 행 사용)
  - 해당 TYPE이 없으면 해당 블록은 생략.

### 4.2 A/D 동시 존재 시(350, 400) 정책

| 방안 | 설명 |
|------|------|
| **가) A만 표시** | 사각형 축류팬(SLB)은 A타입 프레임과 동일하다고 보고, 치수(A)와 SLF_A_Size.png만 표시. |
| **나) D만 표시** | 제품 구성상 D타입만 해당한다면 치수(D)와 SLF_D_Size.png만 표시. |
| **다) A+D 둘 다 표시** | 두 타입 치수 모두 노출, 이미지도 SLF_A_Size.png + SLF_D_Size.png 세로 배치. (환기팬 탭과 동일) |

**권장**: 제품 구조상 SLB가 **A타입 프레임**과 동일하다면 **가) A만 표시**가 단순하고 혼동이 적음. 필요 시 나)·다)로 변경 가능.

---

## 5. 치수 이미지 적용

| 모터 선택 | 치수 데이터 출처 | 표시 이미지 |
|-----------|------------------|-------------|
| **SLB-960** | SLB960_Dim | `SLB_A_Size.png` (기존) |
| **SLB-350, 400** (A만 표시 시) | SLF-350A / SLF-400A | `SLF_A_Size.png` |
| **SLB-350, 400** (D만 표시 시) | SLF-350D / SLF-400D | `SLF_D_Size.png` |
| **SLB-350, 400** (A+D 둘 다 표시 시) | SLF-350A·350D 등 | `SLF_A_Size.png` + `SLF_D_Size.png` |
| **SLB-450, 500, 600, 630, 730** | SLF-450A, 500A, 600A, 630A, 730A | `SLF_A_Size.png` |

- **표시 조건**: 위와 같이 치수 카드가 그려질 때만, 방안 B에 따라 **좌측**에 해당 도면 이미지 표시.

---

## 6. 구현 시 로직 요약

1. **모터 선택값**에서 규격 숫자 추출: `model.match(/SLB-(\d+)/)` → `sizeNum`.
2. **sizeNum === 960**  
   - `slb960Dim`에서 모델 `"SLB-960"` 행 사용.  
   - 치수 카드: 높이(mm), 지름(mm).  
   - 이미지: `SLB_A_Size.png`.
3. **그 외 (350, 400, 450, 500, 600, 630, 730)**  
   - `baseTypeA = "SLF-" + sizeNum + "A"`, `baseTypeD = "SLF-" + sizeNum + "D"`.  
   - `slfADimensions`에 `TYPE === baseTypeA` 행 있으면 → 치수 (A) 카드 + `SLF_A_Size.png` 표시.  
   - `slfDDimensions`에 `TYPE === baseTypeD` 행 있으면 → 치수 (D) 카드 + `SLF_D_Size.png` 표시.  
   - (정책이 A만이면 D 조회/표시 생략.)
4. **매칭되는 치수 없음** (예: 데이터 누락)  
   - 치수 카드·이미지 미표시.

---

## 7. 문서·데이터 연계

- **치수이미지_표시조건.md**  
  - 모터 섹션에 “SLB-960 → SLB960_Dim + SLB_A_Size.png”, “SLB-350 등 → 대응 SLF 치수 + SLF_A_Size.png / SLF_D_Size.png” 조건 추가.
- **재원표**  
  - SLB 전용 치수 시트를 350/400 등으로 확장하지 않고, “모터는 동일 규격 SLF 치수 참조”로 정리해 두면 이후 데이터 일원화에 유리함.

이 방안대로 적용 시, 모터(SLB-350 등)는 환기팬(SLF-350 등)과 동일한 치수 표기·이미지를 사용하게 됩니다.
