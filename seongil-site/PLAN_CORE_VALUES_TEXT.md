# 5대 핵심가치 카드 – 텍스트 자연스럽게 배치하는 계획

## 1. 현재 구조

- **카드**: 아이콘(이모티콘/이미지) → 제목(h3) → 설명(p)
- **레이아웃**: `text-align: center`, 카드 너비 `min(320px, 100%)`, `line-height: 1.6`(카드) / `1.7`(설명)
- **여백**: 아이콘 `margin-bottom: 1rem`, 제목 `margin-bottom: 0.5rem`

---

## 2. 자연스러운 배치를 위한 원칙

| 항목 | 목표 |
|------|------|
| **시각적 위계** | 아이콘 → 제목 → 설명 순서가 한눈에 들어오도록 간격·크기 정리 |
| **가독성** | 설명 문단의 줄 길이·줄 간격을 읽기 좋게 |
| **여백 리듬** | 아이콘–제목–설명 사이 간격을 일정 비율로 통일 |
| **정렬** | 제목은 강조용 중앙, 설명은 읽기 편한 정렬(중앙 유지 또는 왼쪽 정렬 선택) |

---

## 3. 배치 옵션

### 옵션 A: 중앙 정렬 유지 + 여백·타이포만 조정 (변경 최소)

- **아이콘 ↔ 제목**: `1rem` → `1.25rem` 으로 약간 확대.
- **제목 ↔ 설명**: `0.5rem` → `0.75rem` 으로 확대.
- **설명**: `line-height: 1.75`, `max-width: 26em`(카드 안에서만) 로 줄 길이 제한 → 한 줄당 글자 수 적당히 유지.
- **카드 패딩**: `1.5rem` → `1.75rem` 으로 조금 확대해 텍스트가 답답하지 않게.

**효과**: 구조는 그대로 두고, 간격과 줄맞춤만으로 자연스럽게 보이게 함.

---

### 옵션 B: 제목만 중앙, 설명은 왼쪽 정렬 (읽기 최우선)

- **카드**: `text-align: center` 유지하되, **설명 영역만** `text-align: left` 적용.
- **설명**: `display: block`, `margin-left: auto`, `margin-right: auto` 로 블록은 가운데 두고, 글은 왼쪽 정렬.
- **설명 최대 너비**: `22em` ~ `26em` 으로 제한해 줄이 너무 길어지지 않게.

**효과**: 제목·아이콘은 “카드 헤더”처럼 중앙, 본문은 일반 문단처럼 왼쪽 정렬로 읽기 편하게.

---

### 옵션 C: 텍스트 블록을 한 덩어리로 (아이콘 / 텍스트 그룹 분리)

- **HTML**: 카드 안을 `value-icon` / `value-text`(또는 `value-body`) 두 블록으로 구분.
  - `value-text` 안에 `h3` + `p` 만 넣음.
- **스타일**:
  - 아이콘: 기존처럼 상단 중앙.
  - `value-text`: 상단 여백 `1.25rem`, 내부 `text-align: center` 또는 옵션 B처럼 제목 중앙·설명 왼쪽.
  - `value-text` 에만 `padding: 0 0.5rem` 등으로 좌우 여백을 주어 아이콘보다 약간 좁게 → 시선이 “텍스트 블록”에 모이게.

**효과**: 아이콘과 텍스트가 시각적으로 분리되어, 텍스트가 한 덩어리로 읽히고 정렬·간격 조정이 쉬움.

---

## 4. 권장 수치 (공통 적용)

- **아이콘 아래**: `1.25rem`.
- **제목 아래**: `0.75rem`.
- **설명**  
  - `line-height`: `1.75` ~ `1.8`  
  - `font-size`: 카드 기본보다 작게 하지 않음(가독성 유지).  
  - 최대 너비: `26em`(카드 내부 기준).
- **카드 패딩**: `1.5rem` ~ `1.75rem` 유지 또는 통일.
- **제목**: `font-size` `1.2rem` 유지, 필요 시 `1.25rem` 까지만 확대해 위계 유지.

---

## 5. 구현 순서 제안

| 단계 | 작업 | 비고 |
|------|------|------|
| 1 | 옵션 A 적용 | 여백·line-height·설명 max-width만 CSS로 조정 |
| 2 | 필요 시 옵션 B 적용 | 설명에만 `text-align: left` + max-width |
| 3 | 더 구조를 나누고 싶으면 옵션 C | app.js에서 `value-text` 래퍼 추가 후 CSS 적용 |

**우선 적용 권장**: 옵션 A만 적용해도 “자연스럽게 배치”된 느낌을 주기 좋고, 변경 범위가 작음.  
이후 읽기 편함을 더 높이고 싶으면 옵션 B, 카드 구조를 명확히 나누고 싶으면 옵션 C를 추가하면 된다.
